# Инструкция по деплою

## Быстрый деплой на популярные платформы

### 1. GitHub Pages

1. Создайте репозиторий на GitHub
2. Загрузите все файлы проекта
3. Перейдите в Settings → Pages
4. Выберите ветку `main` и папку `/ (root)`
5. Сайт будет доступен по адресу: `https://ваш-username.github.io/название-репозитория`

### 2. Netlify

1. Зарегистрируйтесь на [netlify.com](https://netlify.com)
2. Перетащите папку проекта в Netlify Drop
3. Или подключите GitHub репозиторий
4. Сайт будет автоматически задеплоен

**Настройки:**
- Build command: (оставить пустым)
- Publish directory: `.` (корневая папка)

### 3. Vercel

1. Установите Vercel CLI: `npm i -g vercel`
2. В папке проекта выполните: `vercel`
3. Следуйте инструкциям
4. Или подключите через веб-интерфейс на [vercel.com](https://vercel.com)

### 4. Cloudflare Pages

1. Зарегистрируйтесь на [cloudflare.com](https://cloudflare.com)
2. Перейдите в Pages → Create a project
3. Подключите GitHub репозиторий или загрузите файлы
4. Build settings: (оставить пустым, это статический сайт)

### 5. Обычный хостинг (cPanel, FTP)

1. Загрузите все файлы через FTP или файловый менеджер
2. Убедитесь, что `index.html` в корневой директории
3. Если используется Apache, файл `.htaccess` будет применен автоматически
4. Проверьте права доступа к файлам (644 для файлов, 755 для папок)

## Важные моменты перед деплоем

✅ **Проверьте:**
- Все изображения загружены в папку `Images/`
- Все иконки загружены в папку `icons/`
- Пути к файлам относительные (уже настроено)
- Мета-теги содержат правильный домен (blackwood.uz)

## После деплоя

1. Проверьте работу сайта на всех устройствах
2. Протестируйте все ссылки и кнопки
3. Проверьте загрузку изображений
4. Убедитесь, что модальное окно работает
5. Проверьте SEO мета-теги через [Google Search Console](https://search.google.com/search-console)

## Обновление сайта

После изменений просто загрузите обновленные файлы на хостинг. 
Если используете Git, сделайте commit и push - сайт обновится автоматически.
